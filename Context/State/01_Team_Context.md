# 전략추진실 Team Context

## 이 팀은 무엇을 하는 곳인가

전략추진실은 CSO(이창준) 직속. 잡코리아/알바몬/잡플래닛의 전략 수립과 실행을 지원한다.
인턴은 정형 업무(분석, 장표, 비용처리)를 실행하고, 창준님은 방향과 판단을 내린다.

## 팀 구성 (2026.03 기준)

| 이름 | 역할 | 주 업무 |
|------|------|---------|
| 이창준 | CSO, 전략추진실 리드 | 방향 설정, 의사결정, 업무 요청 |
| 주호연 | 인턴 | Placement, 장표, 비용처리, AX 전환 |
| 임성욱 | 인턴 | Macro, 장표, BCG 서포트 |
| 임한솔 | 인턴 | 품의 자동화, Git 통합, 구매품의 |

## 창준님이 원하는 일하는 방식

### 1. Context만 받고 스스로 답 찾기
- 창준님은 문제와 맥락을 준다. 정답을 주지 않는다.
- 채택되는 결과물이 창준님이 말한 예시와 같을 수도, 다를 수도 있다. 그렇게 되기를 원한다.
- "이렇게 하면 될까요?"보다 "이렇게 했습니다. 이 부분이 맞는지 확인 부탁드립니다."

### 2. 막히면 빠르게 Align
- 큰 문제를 받을 때가 많다. 명확한 답을 못 내릴 수 있다.
- "아무거나 가져가야지"로 시간 쓰는 것을 가장 싫어한다.
- 어디서, 왜 막혔고, 어떻게 할 것인지를 슬랙/아웃룩으로 빠르게 공유한다.

### 3. AI 적극 활용, 생각하면서 일하기
- 바쁘게 일하는 게 아니라, 자동화할 수 있는 것은 AI에 위임하고 생각하면서 일하기를 원한다.
- Claude Code, Genspark을 주 도구로 사용한다.
- "코드 위에서 git push/pull 하며 협업해야 한다"는 방향을 제시했다 (2026.03.20).

### 4. 데이터 정확성
- 장표의 숫자는 반드시 더블체크한다.
- 데이터 연결(엑셀 링크)을 금지한다. 숫자를 직접 확인하고 넣는다.
- 소수점, RMS 계산, 그래프 단위가 틀리면 즉시 지적한다.

### 5. 우선순위
1. 창준님 긴급 요청 (슬랙/아웃룩으로 직접 오는 것)
2. 정기 업무 (Macro, Placement — 분기별 데드라인)
3. AI 전환, 사업 공부 (워크스페이스 개발, 자동화 고도화)

## 우리의 업무 원칙

### 정형 업무는 시스템이 한다
- 반복되는 업무(Placement 파이프라인, Concur 처리, 장표 번역)는 에이전트가 처리한다.
- 사람은 결과를 검수하고, 예외를 판단한다.
- "다음 인턴이 와도 워크스페이스가 설명해준다"가 목표.

### 암묵지는 마크다운으로 남긴다
- 슬랙에서 구두로 전달되던 노하우를 Context/Topic/Skill 파일로 문서화한다.
- 인턴이 바뀌어도 지식이 유실되지 않는다.
- 파일은 Git으로 관리한다. 변경 이력이 감사 추적이 된다.

### 입력은 최소, 출력은 검수만
- "26Q1 해줘" 한 마디로 파이프라인이 돈다.
- 에이전트가 못 하는 판단(미분류 채널 분류, 장표 스토리라인)만 사람이 개입한다.
- 결과물은 대시보드에서 프리뷰하고 승인/수정한다.

### 모르면 물어본다, 단 맥락과 함께
- "이거 어떻게 해요?" ❌
- "A를 하려고 하는데, B 때문에 막혔습니다. C 방향으로 가도 될까요?" ✅
- 에이전트도 마찬가지. 판단이 필요하면 사람에게 맥락과 함께 질문한다.

## 업무 흐름

```
슬랙 업무 요청 → Active Projects에 등록
→ 라우팅 (에이전트 자동 / 사람 판단)
→ 에이전트: Skill 실행 → 결과 프리뷰 → 검수 → 완료
→ 사람: Context 로드 → 작업 → 보고 → 피드백 반영
→ 완료 시 Active Projects에서 해제
```
