# Topic: Placement Survey

## 개요
잡코리아(JK)와 알바몬(AM)의 분기별 채용 서베이. 엠브레인에 위탁하여 Raw 데이터를 받고, 분류→RMS 계산→PPT 장표까지 생성한다.

## 타임라인
- 분기 시작 후 약 2~3주에 Raw 데이터 도착 (엠브레인 → 창준님 → 인턴)
- Raw 도착 후 1~2주 내 분석 + PPT 완료 목표
- 분기: Q1(1~3월), Q2(4~6월), Q3(7~9월), Q4(10~12월)

## 절차 (Step by Step)

### 1. Raw 데이터 수령
- 엠브레인 문주원님이 창준님께 전달 → 인턴에게 공유
- JK Raw, AM Raw 각각 별도 파일

### 2. JK 파이프라인
```
run_jk.py → R_통합 + 미분류 리포트
```
- 분류표(Channel 294개 기준)로 Raw를 분류
- 미분류 발생 시: 새 채널이 등장한 것 → 분류표에 수동 추가 필요
- 미분류 처리 후 재실행

### 3. AM 파이프라인
```
run_am.py → R_통합 + 미분류 리포트
```
- AM용 분류표 기준 (create_am_bunryupyo.py로 추출, 130채널)
- Population weight 적용 (JK와 다른 점)

### 4. RMS 계산
```
calc_rms.py → JK RMS 14시트
calc_rms_am.py → AM RMS 14시트
```
- 검증: 기존 Excel과 소수점 4자리까지 일치 확인

### 5. PPT 생성
```
gen_ppt.py → PPT Summary 6슬라이드 차트 업데이트
```

### 6. 결과 보고
- 창준님께 PPT + 주요 시사점 공유
- 셰어포인트 "3. JK·AM Job Placement Analysis" 폴더에 아카이브

## 주의사항 (암묵지)
- **미분류 처리가 핵심 병목** — 새 채널(커리어리, 탤런트뱅크 등)이 매 분기 등장함
- **Cubicle/Seg 분류** — JK는 Cubicle 기준, AM은 Seg 기준. 혼동 주의
- **RMS 소수점** — 창준님이 소수점 4자리까지 확인. 반올림 방식 통일 필수
- **데이터 연결 금지** — PPT 숫자는 엑셀 링크가 아니라 값으로 직접 입력
- **분류표 수정 후 반드시 재실행** — 부분 수정 후 전체 파이프라인 다시 돌려야 정합성 보장
