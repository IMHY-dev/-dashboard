# Topic: Concur 송장 처리

## 개요
모든 대금 지급은 Concur를 통해 처리한다. 세금계산서가 발행되면 연동 송장이 자동 생성되며, 이를 마무리해서 제출하는 것이 기본 흐름.

## 기본 프로세스

### 1. 세금계산서 발행 확인
- 업체가 세금계산서 발행 → 1~2일 후 Concur에 연동 송장 생성
- 세금계산서는 보통 창준님을 담당자로 발행됨

### 2. 담당자 변경 요청
- 재무회계팀 박은미님에게 "Concur 담당자 변경 요청"
- 변경되면 Outlook으로 "송장이 지정됨" 메일 수신
- 메일 내 "송장 처리 중 링크" 클릭

### 3. 송장 마무리
- 송장 상세 정보 작성 (업무별로 다름)
- 항목별 명세 요약 작성
- 첨부파일: 거래명세서, 세금명세서, 예산품의 (3가지 기본)

### 4. 제출
- 검토 후 제출

## 수기 작성이 필요한 경우
- 세금계산서 발행 전 급하게 대금 지급이 필요한 경우에만
- 이 경우에도 연동 송장이 나중에 생성되므로 삭제 필요 → 번거로움
- 가능하면 세금계산서 발행 후 연동 송장으로 처리

## 업무별 Concur 참고

| 업무 | 특이사항 |
|------|----------|

| 기프티콘 (KT알파) | 정산명세서 기반 |

## 예산 이월
- Concur에서 "잔액 부족" → 재무회계팀에서 예산 이월 필요하다고 슬랙 옴
- 전자결재 > 문건등록 > [예산관리] 7)예산배정(이월)
- 첨부: 해당 예산품의 문서

## 주의사항
- 수기 작성보다 연동 송장 처리가 원칙
- 담당자 변경을 먼저 요청 (안 하면 창준님 앞으로 계속 감)
- 재무회계팀 담당자: 박은미님 (Concur), 김세진 팀장님 (엔코라인/ENS)
